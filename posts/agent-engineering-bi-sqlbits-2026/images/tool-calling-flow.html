Tool Calling

How Agents Use Tools

The LLM reads tool schemas from context, selects the right tool, fills in parameters, and validates results

01
Discover
Read available tool schemas from the context window
▶
02
Select
Pick the right tool for this step of the task
▶
03
Parameterise
Fill in required arguments from conversation context
▶
04
Invoke
Execute the tool call and wait for results
▶
05
Validate
Check the result — retry or proceed to next step

A Tool Schema Is a Contract

{
  "name": "query_semantic_model",
  "description": "Execute a DAX query against the Power BI semantic model",
  "parameters": {
    "model_name": { "type": "string", "description": "Name of the semantic model" },
    "dax_query":   { "type": "string", "description": "Valid DAX query to execute" },
    "max_rows":    { "type": "integer", "description": "Maximum rows to return (default 1000)" }
  }
}
// Better descriptions = better tool selection = better results. This is context engineering again.
