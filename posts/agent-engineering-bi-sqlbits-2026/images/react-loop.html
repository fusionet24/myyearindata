ReAct Pattern

The ReAct Loop

Think. Critique. Act. Observe. Repeat until the task is complete.

01
Think
Reason about the task. Break it down. Plan the approach.
▶
02
Self-Critique
Check assumptions. Am I sure? Do I need more info first?
▶
03
Act
Call a tool. Run a query. Make a request. Do something concrete.
▶
04
Observe
Read the result. Did it work? What did we learn? What's next?
↺
Loop until the task is complete — or the agent decides to ask for help
The critical decision: act now with partial info, or ask a clarifying question first?

BI Example: "Show me revenue by region"

Think "I need the sales fact table and the region dimension."
Critique "Wait — billing region or shipping region? The schema has both."
Act Ask the user: "Do you mean billing region or shipping region?"
Observe User says "billing region." Now I have what I need.
Act Execute: SELECT region, SUM(revenue) FROM sales JOIN regions ON billing_region_id...
Observe Results returned. 5 regions, totals look reasonable. Task complete.
